# Workflow: Respaldo seguro en Git 🛡️

Como **DevOps Master**, ejecutaré la rutina de respaldo seguro para garantizar que no se pierda nada del código actual.

// turbo-all
1. Reviso el status de git para ver qué archivos han cambiado.
2. Agrego todos los archivos nuevos y modificados al área de preparación con `git add .`
3. Genero un commit de autoguardado usando la fecha y hora actual para el mensaje.
4. Envio (push) los cambios a tu rama remota en el repositorio.
5. Reporto el éxito del respaldo mediante un resumen en pantalla.
